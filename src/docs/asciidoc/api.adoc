= REST API Reference

Base URL: `http://localhost:8080`

== GET /

Returns the list of available endpoints.

.Response `200 OK`
[source,json]
----
{
  "application": "smsrouter",
  "endpoints": [
    { "method": "POST", "path": "/api/sms/send",     "description": "Submit an SMS for routing" },
    { "method": "GET",  "path": "/actuator/health",  "description": "Application health" },
    { "method": "GET",  "path": "/actuator/metrics", "description": "Application metrics" }
  ]
}
----

== POST /api/sms/send

Submit an SMS message for routing and delivery.

=== Request body

[cols="1,1,3", options="header"]
|===
| Field | Type | Description
| `from` | String | E.164 sender phone number (e.g. `+33612345678`)
| `to`   | String | E.164 recipient phone number (e.g. `+447911123456`)
| `text` | String | SMS body
|===

[source,bash]
----
curl -s -X POST http://localhost:8080/api/sms/send \
  -H "Content-Type: application/json" \
  -d '{"from":"+33612345678","to":"+447911123456","text":"Hello"}'
----

=== Response `202 Accepted`

[cols="1,1,3", options="header"]
|===
| Field | Type | Description
| `id`        | UUID   | Unique message identifier
| `status`    | String | `DELIVERED` or `FAILED`
| `operator`  | String | Operator that handled the SMS
| `latencyMs` | long   | Delivery latency in milliseconds
|===

[source,json]
----
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "status": "DELIVERED",
  "operator": "EE",
  "latencyMs": 32
}
----

=== Response `422 Unprocessable Entity`

Returned when no operator matches the destination prefix.

[source,json]
----
{
  "error": "Unroutable",
  "phoneNumber": "+99999999999"
}
----

NOTE: An `SMS_FAILED` event with `reason: UNROUTABLE` is also published to `sms.dlq`.
